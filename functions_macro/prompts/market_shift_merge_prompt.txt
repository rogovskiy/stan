You are comparing market shift entries that were grouped by similar type, category, and normalized headline. Your task is to decide whether they are duplicates (same underlying driver) and, if so, how to merge them into one canonical shift.

**Input – shifts in this cluster (each has document id, headline, summary, and other fields):**

{shifts_json}

**Instructions:**
1. Compare the shifts. If they describe the same underlying market driver (same cause, same narrative), treat them as duplicates and output a merge decision.
2. If they are NOT the same driver (different events or angles), output noMerge: true and do not fill merge fields.
3. When merging:
   - Pick one document id to keep as the canonical record (canonicalId). Prefer the one with the highest momentumScore, or oldest firstSeenAt if scores are similar.
   - Choose the best headline and summary (canonicalHeadline, canonicalSummary)—you may combine or pick the clearest.
   - Merge articleRefs from all duplicates into one list, deduplicated by url.
   - Merge channelIds: union of all channelIds from the cluster.
   - For timeline: set keepTimelineFrom to the id of the shift that has the best (most complete) timeline, or the canonical id if equal.
   - List all other document ids in this cluster that should be deleted (mergeIntoCanonical)—every id in the cluster except canonicalId.

**Output format – valid JSON only, no other text:**

If merging:
{
  "noMerge": false,
  "canonicalId": "<document id to keep>",
  "canonicalHeadline": "...",
  "canonicalSummary": "...",
  "mergeIntoCanonical": ["<id1>", "<id2>"],
  "mergedArticleRefs": [{"url": "...", "title": "...", "source": "...", "publishedAt": "..."}],
  "mergedChannelIds": ["EQUITIES_US", "RATES_SHORT"],
  "keepTimelineFrom": "<document id whose timeline to keep>"
}

If NOT merging (different drivers):
{
  "noMerge": true
}
